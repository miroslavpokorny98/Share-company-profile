ISP Map Visualization — 5 Approaches

Different ways to show Share's ISP partner network on the website. Ordered from simplest to most complex. All demos use Kenya as the focus area — the same patterns extend to pan-African scale.

1

Static Dot Map

Simple — CSS only, no JS required

ISP partners as colored dots placed on a map. Each dot has a hover state showing name, location, and user count. Coverage is implied by dot size. This is the baseline — fast to build, fast to load, works perfectly on mobile.

Strengths

  • Near-zero load time — pure SVG + CSS
  • Perfect mobile performance
  • Easy to update via CMS (Sanity)
  • Accessible — works with screen readers
  • Can add dots dynamically as partners join

Limitations

  • Doesn't convey the "alive, growing" feeling
  • No sense of data flowing through the network
  • Coverage areas are only implied, not shown
  • Feels static — may not match immersive homepage

Tech Stack

  • SVG inline or loaded from Sanity
  • CSS hover states + transitions
  • No JavaScript required (optional for tooltips)
  • ~5KB total asset size
  • Build time: 1-2 days
2

Animated Coverage Bubbles

Simple — GSAP, lightweight

ISP dots with pulsing coverage radius animations. Each ISP has a breathing glow that expands and contracts, showing their active coverage area. Connection lines from each ISP to Share's backbone pulse with light. Feels alive without being heavy.

Strengths

  • Feels alive — coverage areas breathe and pulse
  • Shows explicit coverage radius per ISP
  • Data pulses suggest network activity
  • Still very lightweight (~15KB + GSAP)
  • Great mobile performance

Limitations

  • Coverage circles are approximate, not real data
  • Gets cluttered with 20+ ISPs
  • Flat map doesn't convey continental scale

Tech Stack

  • SVG + GSAP for breathing/pulse animations
  • ISP data from Sanity CMS or API
  • ~50KB total (including GSAP)
  • Build time: 2-3 days
3

Network Topology Graph

Medium — GSAP + interaction

Shows the actual network hierarchy: subsea cables at the edges, Share data centers in the middle tier, ISP partners as the outer ring. Connections show the real topology — which ISP connects through which data center. Animated data flowing through the network paths. Click a node to highlight its connections.

Strengths

  • Shows real network architecture — ISPs see exactly how they connect
  • Three tiers visible: subsea → Share DCs → ISPs
  • Answers "how does Share work?" visually
  • Data pulses show the network is active
  • Can highlight individual paths on hover/click

Limitations

  • Not geographic — doesn't show WHERE ISPs are
  • More diagram than map — feels technical
  • Harder to understand for non-technical visitors
  • Needs careful layout to avoid visual clutter

Tech Stack

  • SVG + GSAP for pulse animations
  • Optional: click interaction for path highlighting
  • ISP/DC data from API or Sanity
  • ~60KB total
  • Build time: 3-5 days
4

Animated Data Flow Map

Medium — GSAP + canvas/particles

Combines the geographic map with live-feeling data flows. ISPs sit on real positions, connected to Share's backbone via fiber routes. Animated particles stream along these routes continuously — the map feels like a living organism. Particle density and speed indicate traffic volume. This is the sweet spot between visual impact and technical feasibility.

Strengths

  • Geographic + data flow — best of both worlds
  • Particles make the network feel alive and real-time
  • Shows actual fiber routes, not just point-to-point
  • Traffic volume visible through particle density
  • Can pull real throughput data from Share's API

Limitations

  • Particle rendering needs optimization for mobile
  • Real fiber route data needed for accuracy
  • More complex to maintain and update
  • Flat map still limits the "wow" factor

Tech Stack

  • SVG for map + Canvas overlay for particles
  • GSAP or requestAnimationFrame for animation
  • Optional: real-time data from Share API
  • ~100KB + optimization for 60fps
  • Build time: 5-7 days
5

3D Interactive Globe

Complex — Three.js / WebGL

Full 3D globe with ISP markers as glowing pins, subsea cables as arcs curving across the ocean surface, and coverage areas as luminous regions. User can rotate, zoom, and click ISPs to see details. Data flows are animated arcs between nodes. This is the reference Google Cloud Infrastructure used — the one Share specifically called out as "what we're doing, but for Africa to the world."

Strengths

  • Maximum "wow" — matches Google Cloud Infrastructure reference
  • Shows Africa in global context (subsea cable arcs to world)
  • Rotation gives sense of real geography and scale
  • Pins + arcs are visually stunning and memorable
  • Conveys "we're a global infrastructure player"

Limitations

  • Heavy — Three.js is ~150KB + textures + geometry
  • Poor performance on low-end Android devices
  • Needs WebGL support (98% of browsers, but slower connections hurt)
  • Much harder to make accessible
  • Needs fallback for non-WebGL devices

Tech Stack

  • Three.js + custom shaders for globe
  • GeoJSON data for Africa outline on sphere
  • GSAP for camera animation, ScrollTrigger optional
  • ~300-500KB total (with lazy loading)
  • Build time: 10-15 days
  • Needs SVG fallback for low-end devices
